# MauiApp 截图指南

## 概述
此文档指导用户如何为审查准备 MauiApp 的界面截图。

## 准备工作

### 1. 环境配置
```bash
# 安装 MAUI 工作负载
dotnet workload install maui-android

# 进入项目目录
cd ZakYip.Singulation.MauiApp

# 构建项目
dotnet build -f net8.0-android
```

### 2. 启动服务
在截图前，确保 Host 服务正在运行：
```bash
cd ../ZakYip.Singulation.Host
dotnet run
```

### 3. 运行 MauiApp
- 在 Android 模拟器上运行
- 或在真实 Android 设备上运行

## 截图技巧

### Android 模拟器截图
1. 使用模拟器工具栏的相机图标
2. 或使用 `Ctrl + S` (Windows) / `Cmd + S` (Mac)
3. 截图保存在桌面

### 真实设备截图
1. Android: 电源键 + 音量减
2. 通过 USB 连接后，使用 `adb shell screencap` 命令
3. 或使用 Android Studio 的 Device File Explorer

### 截图质量要求
- **分辨率：** 至少 1080x1920 或设备原生分辨率
- **格式：** PNG（推荐）或 JPG
- **命名：** 按照上述建议的文件名命名
- **清晰度：** 确保文字和图标清晰可读

## 提交方式

### 方式一：GitHub Issue
1. 在 GitHub 仓库中创建新 Issue
2. 标题：`MauiApp UI 审查 - 界面截图`
3. 按顺序上传所有截图
4. 注明任何问题或观察到的异常

### 方式二：Pull Request 评论
1. 在当前 PR 的评论区
2. 使用 Markdown 语法引用截图
3. 格式：
   ```markdown
   ## MauiApp 界面截图
   
   ### 主页面
   ![主页面初始状态](./01-mainpage-initial.png)
   
   ### 设置页面
   ![设置页面](./04-settings-discovering.png)
   
   ...
   ```

### 方式三：直接反馈
1. 将截图打包为 ZIP 文件
2. 命名：`mauiapp-screenshots-YYYYMMDD.zip`
3. 通过其他渠道发送给开发者

## 审查要点

当您审查截图时，请关注以下方面：

### UI 设计
- [ ] 图标和文字的视觉分离是否清晰

- [ ] 自动发现是否真正自动启动

- [ ] 空状态提示是否友好

- [ ] UDP 服务发现是否工作
- [ ] API 地址配置是否正确
- [ ] 所有按钮功能是否正常
